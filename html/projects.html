University Group Project - My model Cars

my second year group project involved designing an interactive mobile application to enhance engagement for model car collectors


we designed a top trumps style game where users were able to scan a collection of real life model cars and play with them in game


being team lead meant that i was leading client meetings, orgainsing sprints and milestones using jira and presenting the final project in front of a room of acadameics and industry leaders


The key development areas within the project were:

  • unity development: the game was written entirely in unity and then deployed onto iphone
  • machine leaning pipeline: we implmented a three stage pipeline (blur detection -> car detection -> car prediction)
  • model car collection system: when a user scanned in a model car, it was added to a garage or collection to prevent having to rescan
  • uniquely designed 3d models: hand crafted 3d models were designed by us in maya autodesk


MNIST-Neural-Network

An implementation of training a neural network using backpropagation from scratch


The key development features of the project are:


  • supports any architecture: the user can select any architecture, i.e. (123,2,3)
  • tested using mnist digits: the network was tested using the mnist digits dataset
  • entirely in python: the code is implemented entirely in python mainly using numpy for efficient matrix manipulation

Brief code explanation:


  • training loop: for n epochs we perform a forward pass (compute activations), calculate loss (mse) and then a backwards pass to calculate gradients and update weights
  • backpropagation backpropagation involves computing the gradients of the loss with respect to the weights. These gradients are then used to adjust the weights in an opposite direction to minimise loss
